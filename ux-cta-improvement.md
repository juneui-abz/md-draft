# 공구 상세 CTA 개선

## 1. 목적

공구 상세 하단 CTA가 상태별로 일관성이 없고, 라이팅이 약해 사용자 의사결정이 분산됨.

- 상태별 일관된 레이아웃 + 명확한 라이팅
- 핵심 1개 액션에 CTA 집중
- 종료 공구도 단절이 아닌 다음 액션으로 연결

---

## 2. 검증하려는 가설

> 컨텍스트: DAU 20 전후, 마켓핏 탐색 단계. 통계적 유의성보다 **행동 시그널/패턴 발견**이 목표.
>
> 공통 판단 기준: 발생 증가 + 사용자 다양화 → 가설 살아있음 / 정체 또는 감소 → 가설 약함

### 가설 1: 찜이 저장/탐색 도구로 사용되는가?

**가설**: 사용자는 알림 외에도 공구를 "나중에 다시 보기" 위해 찜을 사용한다.

**측정 이벤트**

| # | 이벤트 | 의미 |
|---|---|---|
| 1 | open + closed 공구 찜 발생 (upcoming 제외) | 알림 무관 저장 행동 |
| 2 | 위 찜 후 7일 내 관심 탭에서 해당 공구 재진입 | 실제 "다시 보기" 행동 |

### 가설 2: 종료 공구 재요청이 의미 있는 액션인가?

**가설**: 사용자는 종료된 공구에서도 다음 행동(재오픈 의향, 다른 공구 탐색)을 할 의지가 있다.

**실험 단계**: MVP — 재요청 = 알림 신청으로 임시 정의.

**측정 이벤트**

| # | 이벤트 | 의미 |
|---|---|---|
| 1 | closed에서 [공동구매 재요청] 탭 | 재요청 의향 |

### 배포 전 baseline

좋아요 수, 좋아요 후 관심 탭 재진입율, 종료 공구 진입 수, CTA 클릭율 — 변경 전 측정해두기.

---

## 3. 주요 변경 근거

### upcoming은 URL 분기 없이 "공동구매 준비중" disabled로 통일
- "공동구매 보기"는 들어가도 못 사니까 기대 불일치 → 이탈
- "준비중" disabled가 상태 메시지 명확
- CTA 집중도 상승 (찜에 올인)
- URL 있는 공구의 정보 확인 니즈는 상세 콘텐츠 영역의 캡션 요약으로 커버 가능

### open은 "할인가로 구매하기"로 전환 유도 강화
- "공동구매 보기" → 중립적, 약한 라이팅
- "할인가로 구매" → 가치 제안 + 행동 명시

### closed → "공동구매 재요청"
- 비활성 텍스트 단절 → 능동적 액션으로 전환
- 사용자 수요 데이터 수집

### 인스타 버튼 제외
인스타그램 홈으로만 보내서 탐색이 불편하고, 상세 정보 영역에 이미 인스타 링크가 있어 제거.
