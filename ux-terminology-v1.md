# 오예 앱 UX 개선 기획 — 1단계: 용어 통일

## 배경

### 문제

하나의 동작(구독)에 UI상 3가지 용어가 혼재:

| 용어 | 쓰이는 곳 |
|---|---|
| 좋아요 | 크리에이터 버튼, 홈 서브탭, 취소 다이얼로그, 토스트, 로그인 유도, accessibility |
| 관심 목록 | GNB 탭 라벨, 공구 상세 버튼, 토스트 링크 |
| 알림 받기 | 키워드 검색 결과 버튼 |

에러 메시지 2곳에서 "구독"이 노출되지만 (에러 상태 한정), 코드 내부(subscription, API)는 사용자에게 보이지 않으므로 주요 문제는 위 3가지.

한 화면 안에서도 충돌 발생:
- 토스트: "**좋아요**를 눌렀어요" + "**관심 목록** 보기"
- 공구 상세: 버튼은 "**관심 목록** 추가" → 취소 시 "**좋아요** 취소"
- 검색: 키워드는 "**알림 받기**", 크리에이터/공구는 하트 아이콘(좋아요)

### 히스토리

- 원래 "알림받기"였다가 → 액션이 헤비하게 느껴져서 "좋아요"로 변경
- 좋아요로 바꾸면서 3가지 서로 다른 성격의 행동이 하나로 뭉뚱그려짐

### 구독 대상별 실제 성격

| 대상 | 현재 용어 | 실제 성격 |
|---|---|---|
| 크리에이터 | 좋아요 | 팔로우 (지속적 관계, 새 공구 알림) |
| 공동구매 | 관심 목록 추가 | 오픈 알림 (일회성, 시점 알림) |
| 키워드 | 알림 받기 | 조건 구독 (매칭 시 알림) |

| 키워드 | **알림받기** | **알림받는 중** | 🔔 벨 (변경) |

### GNB / 네비게이션

| 요소 | 현재 | 변경 |
|---|---|---|
| GNB 3번째 탭 | "관심 목록" (❤️) | **"관심" (❤️)** — 축약만, 구조 유지 |

- 크리에이터 ❤️ 유지: ZVZO도 하트 + "팔로잉" 조합 사용. 변경 최소화.
- 공구/키워드 🔔 변경: "알림받기" 텍스트와 벨 아이콘이 일관. 크리에이터(❤️)와 자연스럽게 구분.

## 경쟁사 레퍼런스

### 위시버니
- 크리에이터: "팔로우" → 팔로우 직후 바텀시트로 "알림받기" 유도
- 바텀시트: 푸시 미리보기 이미지 + "팔로우한 셀러의 공구를 놓치지 마세요" + [알림받기] CTA + "나중에 할게요"
- 팔로우와 알림이 2단계로 분리됨

### ZVZO
- 크리에이터: "팔로우/팔로잉" (❤️ 하트 아이콘)
- GNB: 홈 — 콘텐츠 — **관심** (❤️) — 마이
- 알림: 설정 페이지에서 카테고리별 토글 (찜 브랜드 소식, 크리에이터 소식, 신규 딜, 마감 딜, 카테고리 등)
- 팔로우와 알림 설정이 완전 분리

### 기타

| 앱 | 구독 액션 | 구독 목록 탭 | 알림 수신함 |
|---|---|---|---|
| 당근 | "관심" (하트) | "관심목록" | "알림" (벨) |
| 쿠팡 | "찜" (하트) + "재입고 알림받기" | 마이 하위 | "알림" (벨) |
| 무신사 | "좋아요" (하트) + "재입고 알림받기" | "좋아요" | "알림" (벨) |
| 네이버 스토어 | "찜" (하트) + "소식받기" (벨) | "찜" | "알림" (벨) |

핵심: "알림받기" 버튼과 헤더 "알림" 수신함을 동시에 쓰는 앱이 다수 (쿠팡, 무신사, 배민, 오늘의집). 문맥으로 자연스럽게 구분됨.

## 로드맵

### 1단계: 용어 통일 (이 문서)
- UI 텍스트 + 아이콘 변경
- 백엔드 변경 없음

### 2단계: 팔로우 / 알림 실제 분리
- 팔로우 시 알림 권한 요청 안 함
- 팔로우 후 알림받기 별도 토글 (위시버니 바텀시트 패턴)
- 백엔드 follow / subscribe 분리
- 알림 설정 페이지 (ZVZO 패턴)

### 3단계: 마이페이지 개편 + GNB 구조 변경
- 마이페이지를 구독 대시보드 (캘린더 시안) 로 개편
- GNB "관심" 탭 제거 → 3탭 구조 (홈 — 검색 — 마이)
- 홈 서브탭 역할 재정의
