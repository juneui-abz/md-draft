# 웹 공구 상세 CTA 개선

## 1. 목적

웹은 앱 설치 유도가 핵심 역할. 현재 모든 상태 CTA가 "이 공구 알림받기"로 통일되어 있어, 상태별 맥락에 맞는 라이팅으로 **앱 설치 전환율 향상** + **이탈 방지**.

- 상태별 맥락 맞는 액션 소구

---

## 2. 검증하려는 가설

> 컨텍스트: 웹은 앱 설치 유도 채널. 통계적 유의성보다 **행동 시그널/패턴 발견**이 목표.
>
> 공통 판단 기준: 발생 증가 → 가설 살아있음 / 정체 또는 감소 → 가설 약함

### 가설: 상태별 맥락 라이팅이 CTA 클릭율을 높인다

**측정 이벤트**

| # | 이벤트 | 의미 |
|---|---|---|
| 1 | 상태별 CTA 클릭 (upcoming / open / closed 각각) | 라이팅 효과 |
| 2 | CTA 클릭 → 앱 스토어 이동 | 설치 유도 진행 |

**검증 방법**: 상태별 CTA 클릭율(= 클릭 / 페이지 진입)을 배포 전 baseline과 비교.

**판단**: 상태별 클릭율 증가 → 라이팅 효과 있음 / 변화 없음 → 미미

### 2차 실험: 탐색 소구 (1차 후 진행)

1차에서 전환율 상승 확인 후, "알림" 대신 "탐색" 소구로 변경 테스트.

| 상태 | 1차 (알림 소구) | 2차 (탐색 소구) |
|---|---|---|
| upcoming | 놓치지 않게 알림받기 | 공구 자세히 보기 |
| open | 공구 할인가로 구매하기 | 유지 |
| closed | 재오픈 시 알림받기 | 비슷한 공구 둘러보기 |

**검증**: 1차(알림) vs 2차(탐색) 전환율 비교 → 오예의 마켓핏이 "알림"인지 "탐색"인지 가늠.

### 배포 전 baseline (3/26~4/9 기준)

| 지표 | 수치 |
|---|---|
| 전체 page_view | 2,394 |
| 공구 상세 page_view | 1,084 (45%) |
| app_link_click | 389 |
| 공구 상세 → 앱링크 전환율 | ~36% |

상태별 분포(upcoming/open/closed)는 미확인 → 이벤트에 상태 태깅 후 추가 측정 필요.

---

## 3. 상태별 CTA

| 상태 | AS-IS | TO-BE |
|---|---|---|
| upcoming | 이 공구 알림받기 | **이 공구 놓치지 않게 알림받기** |
| open | 이 공구 알림받기 | **공구 할인가로 구매하기** |
| closed | 이 공구 알림받기 | **재오픈 시 놓치지 않게 알림받기** |

---

## 4. 주요 변경 근거

### upcoming — FOMO 톤 추가
"알림받기" → "놓치지 않게 알림받기"로 오픈 타이밍을 놓칠 수 있다는 감정 소구.

### open — 혜택 명시로 넛지 강화
"알림받기"는 현재 살 수 있는 상태와 맥락 안 맞음. "할인가" 키워드로 혜택 명시. 앱 버전 CTA와 라이팅 일관성 유지.

### closed — 맥락 맞는 라이팅으로 전환
기존 "이 공구 알림받기"는 종료 공구와 맥락 불일치. "재오픈 알림"으로 상태에 맞게 전환. upcoming과 같은 FOMO 톤으로 일관성 유지.
