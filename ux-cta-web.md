# 웹 공구 상세 CTA 개선

## 1. 목적

웹은 앱 설치 유도가 핵심 역할. 현재 모든 상태 CTA가 "이 공구 알림받기"로 통일되어 있어, 상태별 맥락에 맞는 라이팅으로 **앱 설치 전환율 향상** + **이탈 방지**.

- 상태별 맥락 맞는 액션 소구
- 종료 공구의 이탈 포인트를 재방문 루프로 전환
- 탭 시 앱 스토어 → 설치 후 공구 상세 딥링크 이동 (앱 내 동일 액션으로 연결)

---

## 2. 상태별 CTA

| 상태 | AS-IS | TO-BE |
|---|---|---|
| upcoming | 이 공구 알림받기 | **이 공구 놓치지 않게 알림받기** |
| open | 이 공구 알림받기 | **공구 할인가로 구매하기** |
| closed | 이 공구 알림받기 | **재오픈 시 놓치지 않게 알림받기** |

---

## 3. 주요 변경 근거

### upcoming — FOMO 톤 추가
"알림받기" → "놓치지 않게 알림받기"로 오픈 타이밍을 놓칠 수 있다는 감정 소구.

### open — 혜택 명시로 넛지 강화
"알림받기"는 현재 살 수 있는 상태와 맥락 안 맞음. "할인가" 키워드로 혜택 명시. 앱 버전 CTA와 라이팅 일관성 유지.

### closed — 이탈 포인트를 재방문 루프로
기존엔 이탈 포인트였던 영역에 재오픈 알림 CTA 추가. upcoming과 같은 FOMO 톤으로 일관성 유지.

---

## 4. 딥링크 연결

모든 웹 CTA는 앱 스토어 → 설치 후 해당 공구 상세 딥링크로 이동. 앱 내에서 동일한 목적의 액션으로 자연스럽게 연결되어 사용자 기대와 일치.

| 웹 CTA | 앱 도착 후 액션 |
|---|---|
| 이 공구 놓치지 않게 알림받기 | 공구 상세 → 찜하기 |
| 공구 할인가로 구매하기 | 공구 상세 → 할인가로 구매하기 |
| 재오픈 시 놓치지 않게 알림받기 | 공구 상세 → 공동구매 재요청 |
