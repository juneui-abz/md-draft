# 오예 앱 UX 개선 기획 — 관심(좋아요)과 알림 분리

---

## 1. 현재 상태

### 한 액션에 두 행동이 묶여있음

```
[좋아요] 누름 = 관심 목록 저장 + 푸시 알림 구독 (동시에 발생)
```

| 대상 | 액션 | 저장 | 푸시 | 분리 가능? |
|---|---|---|---|---|
| 크리에이터 | "이 크리에이터 좋아요" | 관심 목록에 추가 | 새 공구 오픈 시 푸시 | 현재 불가 |
| 공동구매 | "관심 목록 추가" | 관심 목록에 추가 | 공구 오픈 시 푸시 | 현재 불가 |
| 키워드 | "{키워드} 알림 받기" | 관심 목록에 추가 | 매칭 공구 등장 시 푸시 | 현재 불가 |

### 히스토리

- 원래 "알림받기"였다가 → 액션이 헤비하게 느껴져서 "좋아요"로 변경
- 좋아요로 바꾸면서 3가지 서로 다른 성격의 행동이 하나의 용어로 뭉뚱그려짐

### 용어 혼재

같은 동작에 3가지 용어가 UI에 혼재:

| 용어 | 쓰이는 곳 |
|---|---|
| 좋아요 | 크리에이터 버튼, 홈 서브탭, 취소 다이얼로그, 토스트, 로그인 유도, accessibility |
| 관심 목록 | GNB 탭 라벨, 공구 상세 버튼, 토스트 링크 |
| 알림 받기 | 키워드 검색 결과 버튼 |

한 화면 안에서도 충돌:
- 토스트: "**좋아요**를 눌렀어요" + "**관심 목록** 보기"
- 공구 상세: 버튼은 "**관심 목록** 추가" → 취소 시 "**좋아요** 취소"
- 검색: 키워드는 "**알림 받기**", 크리에이터/공구는 하트 아이콘(좋아요)

에러 메시지 2곳에서 "구독"이 노출 (에러 상태 한정). 코드 내부(subscription, API)는 사용자에게 보이지 않음.

### 화면별 현재 텍스트

**크리에이터 상세**

| 요소 | 구독 전 | 구독 후 |
|---|---|---|
| 프로필 버튼 | "이 크리에이터 좋아요" | "좋아요" |
| 하단 푸터 버튼 | "이 크리에이터 좋아요" | "좋아요" |

**공구 상세**

| 요소 | 구독 전 | 구독 후 |
|---|---|---|
| 하단 푸터 버튼 | "관심 목록 추가" | "관심 목록 추가됨" |
| 카운트 텍스트 | "좋아요 N명이 관심 갖는 중" | - |

**홈**

| 요소 | 텍스트 |
|---|---|
| 서브탭 | "추천" / "좋아요" |
| 추천 크리에이터 버튼 | "좋아요" |
| 빈 상태 | "좋아요 중인 크리에이터가 없어요 😭" |
| 빈 상태 (캠페인) | "구독 크리에이터의 활성 캠페인이 없어요" |

**검색**

| 요소 | 구독 전 | 구독 후 |
|---|---|---|
| 크리에이터 결과 | 하트 (outline) | 하트 (filled) |
| 공구 결과 | 하트 (outline) | 하트 (filled) |
| 키워드 버튼 | "{키워드} 알림 받기" | "좋아요" |

**다이얼로그**

| 상황 | 텍스트 |
|---|---|
| 로그인 유도 | "좋아요를 누르려면 로그인해주세요" |
| 크리에이터 취소 | "좋아요 취소 / 이 크리에이터의 좋아요를 취소할까요?" |
| 공구 취소 | "좋아요 취소 / 이 공동구매의 좋아요를 취소할까요?" |

**토스트**

| 텍스트 |
|---|
| "좋아요를 눌렀어요. 공동구매가 시작되면 알려드릴게요." |
| "관심 목록 보기" (링크) |

**GNB / 기타**

| 요소 | 텍스트 |
|---|---|
| GNB 3번째 탭 | "관심 목록" (❤️ 하트) |
| 헤더 벨 | "알림" (🔔 벨) |
| Accessibility | "좋아요" / "좋아요 해제" |

## 2. 문제점

### 2-1. 알림 의도 없이 저장만 하고 싶은데 푸시가 옴
- 관심은 있지만 알림까지는 원하지 않는 사용자에게 불필요한 푸시 발생
- 푸시가 많아지면 앱 알림 자체를 꺼버리는 결과로 이어질 수 있음

### 2-2. 알림 권한 허들이 저장을 막음
- 좋아요 누르려면 알림 권한 동의 필요 (`guardBeforeSubscribe`)
- 단순 저장만 하고 싶은 사용자가 권한 요청에서 이탈

### 2-3. 끝난 공구가 관심 목록에서 사라짐
- 현재: 서버에서 기간 유효한 공구만 내려줌
- 알림 목적으로는 맞지만, 저장/히스토리 관점에서는 남아있어야 함
- "내가 어떤 공구에 관심 가졌는지" 기록이 유실됨

### 2-4. 푸시 피로도 관리 불가
- 관심 가는 크리에이터/공구가 많을수록 푸시도 비례해서 증가
- 좋아요를 줄이거나, 앱 알림을 끄는 것 외에 제어 수단 없음

## 3. 변경 방향

### 핵심: 저장과 알림을 분리

```
변경 후:
[찜하기/팔로우] = 관심 저장만 (푸시 없음, 알림 권한 불필요)
[알림받기] = 푸시 알림 구독 (별도 액션)
```

### 대상별 액션 설계

| 대상 | 저장 액션 | 저장 상태 | 알림 액션 | 알림 상태 |
|---|---|---|---|---|
| 크리에이터 | **팔로우** | **팔로잉** | **알림받기** (별도) | **알림 중** |
| 공동구매 | **찜하기** | **찜** (filled) | **알림받기** (별도) | **알림 중** |
| 키워드 | — | — | **알림받기** (단일 액션) | **알림받는 중** |

- 크리에이터: "팔로우"가 저장 역할 (인스타 기반 서비스라 자연스러움)
- 공동구매: "찜하기"가 저장 역할
- 키워드: 저장 의미가 없으므로 알림받기 단일 액션 유지, 알림 쪽에서 관리

### 아이콘

| 액션 | 아이콘 |
|---|---|
| 팔로우/찜하기 (저장) | ❤️ 하트 |
| 알림받기 (푸시) | 🔔 벨 |

### GNB / 네비게이션

| 요소 | 현재 | 변경 |
|---|---|---|
| GNB 3번째 탭 | "관심 목록" (❤️) | **"관심" (❤️)** |
| 홈 서브탭 | "추천" / "좋아요" | "추천" / **"팔로잉"** |
| 헤더 벨 (수신함) | 벨 "알림" | 그대로 유지 |

### 관심 탭 구성

**찜하기 영역** — 공동구매 저장/관리

| 현재 | 변경 |
|---|---|
| 구독 중인 공구만 (끝난 공구 숨김) | **찜한 공구 전체** (기본은 유효한 공구만, 필터로 지난 공구까지) |
| 구독 = 알림이라 해제하면 목록에서 사라짐 | 찜 유지 + 알림만 끄기 가능 |

공동구매 관리 기능:
- 일정 순 (D-day 기준 정렬)
- 최근 본
- 추천 리스트

**팔로우 영역** — 크리에이터 관리

**알림 영역** — 키워드 알림은 이쪽에서 관리

## 4. 알림 제어 방식 (확정)

3단계 알림 제어 구조:

| 레이어 | 시점 | 역할 | 제어 단위 |
|---|---|---|---|
| **바텀시트** | 첫 팔로우/찜하기 직후 | 알림 가치 전달 + 유도 | 1회성 |
| **벨 토글** | 상세 화면 | 개별 크리에이터/공구 알림 ON/OFF | 개별 |
| **알림 설정** | 설정 페이지 | 카테고리별 일괄 ON/OFF | 카테고리 |

레퍼런스: 인스타그램 (팔로우 알림 유도 + 개별 벨 + 설정), 위시버니 (바텀시트), ZVZO (설정 페이지), 유튜브 (구독 벨)

### 4-1. 바텀시트 (첫 팔로우/찜하기 시)

```
[팔로우/찜하기] 탭 → 저장 완료 → 바텀시트 올라옴
  - 푸시 미리보기 이미지
  - "새 공구가 열리면 알려드릴까요?"
  - [알림받기] CTA
  - "나중에 할게요" 스킵
```

- 첫 1회만 노출 (같은 대상에 대해)
- "나중에" 선택 시 팔로우/찜만 되고 알림 OFF

### 4-2. 벨 토글 (상세 화면)

```
[팔로우 ❤️] [🔔] 두 버튼이 나란히 배치
  - ❤️: 팔로우/찜하기 (저장)
  - 🔔: 알림 ON/OFF (개별 제어)
```

- 바텀시트에서 "나중에" 했어도 여기서 켜기 가능
- 알림 중인 상태에서 끄기도 가능

### 4-3. 알림 설정 페이지 (설정)

```
설정 > 알림 설정
  - 크리에이터 새 소식 알림 받기      [ON/OFF]
  - 공동구매 오픈 알림 받기           [ON/OFF]
  - D-1 알림                         [ON/OFF]
  - D-day 알림                       [ON/OFF]
```

피로도 관리: 알림을 개별 발송하지 않고 **묶어서 관리** (예: "오늘 오픈 예정 공구 3건" 등으로 다이제스트)

### 사용자 시나리오

```
1. 크리에이터 팔로우 → 바텀시트: "알림도 받을까요?" → [알림받기] or [나중에]
2. 나중에 마음 바뀜 → 크리에이터 상세에서 🔔 벨 탭해서 ON
3. 알림 너무 많음 → 설정 > "크리에이터 새 소식 알림" 전체 OFF
4. D-1에 "내일 오픈 예정 공구 3건" 묶음 알림 수신
5. D-day에 "오늘 오픈하는 공구 2건" 묶음 알림 수신
```
