# 사용자 인터뷰 모집 + 질문 설계

## 1. 구글폼 (모집 + 사전 정보)

**타이틀**: 오예를 더 좋게 만들고 싶어요 ☕️

**안내**: 답변해주신 분께 커피 기프티콘 + 10분 전화 인터뷰 요청드릴 수 있어요

**Q1. 오예를 어떻게 사용하고 계시나요?**
- 공동구매를 찾아보는 사용자로
- 공동구매를 진행하는 크리에이터로

**Q2. 오예를 얼마나 자주 여시나요?**
- 매일
- 주 2~3회
- 주 1회
- 거의 안 열어요

**Q3. 오예를 주로 왜 쓰시나요?** (복수 선택)
- 공구 오픈 알림 받으려고
- 관심 있는 공구 둘러보려고
- 좋아하는 크리에이터 공구 확인하려고
- 할인 공구 찾으려고
- 별다른 이유 없이 습관적으로
- 기타

**Q4. 오예가 없으면 어떨 것 같으세요?**
- 매우 아쉬울 것 같다
- 조금 아쉬울 것 같다
- 별로 상관없다
- 모르겠다

**Q5. 불편하거나 아쉬운 점이 있다면?** (주관식)

**연락처** (필수): 카톡 ID 또는 전화번호

## 2. 인터뷰 대상 선별

- Q1 크리에이터 응답 → 별도 분류
- Q4 "매우 아쉬울 것 같다" → 최우선 (PMF 강한 사용자)
- Q4 "별로 상관없다" → 차순위 (이탈 위험, 왜인지 파악)
- Q2 양극단 (매일 vs 거의 안 열어요) → 다 인터뷰
- Q5 정성껏 쓴 사람 → 우선

## 3. 노출 방법

앱 홈 상단 배너 (1~2주):
"오예를 더 좋게 만들고 싶어요. 1분 설문 참여하기 ☕️"
[참여하기] → 구글폼 링크
[닫기]

## 4. 인터뷰 질문 (10분)

설문 응답 기반으로 심화.

### 도입 (1분)

| # | 질문 | 목적 |
|---|---|---|
| 1 | 오예를 처음 어떻게 알게 되셨어요? | 유입 경로 |

### 사용 동기 (3분)

| # | 질문 | 목적 |
|---|---|---|
| 2 | 마지막으로 오예 열었을 때 뭘 하셨어요? | 과거 행동 기반, 유도 방지 |
| 3 | 찜(좋아요) 누른 적 있으세요? 왜 누르셨어요? | 찜 동기 (알림 vs 저장) |

### PMF 검증 (3분)

| # | 질문 | 목적 |
|---|---|---|
| 4 | 설문에서 오예 없으면 {응답}이라고 하셨는데, 좀 더 얘기해주실 수 있어요? | 설문 답변 심화 |
| 5 | 그러면 뭘로 대체하실 것 같아요? | 대체재 파악 |
| 6 | 오예 전에는 공구 정보를 어떻게 얻으셨어요? | 이전 행동 = 우리가 해결하는 문제 |

### 개선 (2분)

| # | 질문 | 목적 |
|---|---|---|
| 7 | 설문에서 {불편한 점}이라고 쓰셨는데, 구체적으로 어떤 상황이었어요? | 설문 답변 심화 |
| 8 | 오예가 이것만 해주면 좋겠다 하는 게 있으세요? | 사용자 언어로 된 니즈 |

### 마무리 (1분)

| # | 질문 | 목적 |
|---|---|---|
| 9 | 마지막으로 하고 싶은 말씀 있으세요? | 예상 못한 인사이트 |

### 인터뷰 팁

- Q2 "마지막으로 뭘 하셨어요?"가 가장 중요 — 유도 없이 실제 행동
- Q4, Q7은 설문 답변 기반으로 파고듦
- 모든 답변 후 "왜요?"를 2~3번 더 파고들기
- 10분이니까 전부 못 물어도 OK, Q2~Q6이 핵심
